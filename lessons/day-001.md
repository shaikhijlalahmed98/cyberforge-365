## 2. Core Concept — The CIA Triad

Yaar, **cybersecurity** ka full definition pehle samajh le: it's the discipline of protecting **systems, data, and people** from unauthorized access, alteration, destruction, or disruption. Bas. No magic, no hoodies, no green-on-black terminals. Yeh ek **engineering discipline** hai — jaise reliability engineering, lekin adversary ke against.

Ab CIA — yeh acronym kisi agency se related nahi. Yeh hai **Confidentiality, Integrity, Availability**. Har security control duniya mein in teeno mein se kam-az-kam ek ko serve karta hai.

### 🔐 Confidentiality — "Yeh data dekhega kaun?"

Matlab: sirf authorized log ko data dikhna chahiye. Tu apne .NET microservice mein `[Authorize(Roles = "Finance")]` lagata hai na — woh **confidentiality control** hai. Spring Security mein `@PreAuthorize("hasRole('FINANCE')")` — same baat.

**Examples enterprise mein:**
- Database column-level encryption (PII columns)
- TLS on the wire (HTTPS) — network pe koi sniff na kare
- IAM policies — S3 bucket pe `Principal: *` matlab confidentiality dead

**Failure mode:** Data leak. Newspaper headlines. GDPR €20M fines.

```csharp
// .NET example — confidentiality at the API layer
[HttpGet("salary/{employeeId}")]
[Authorize(Roles = "HR,Finance")]   // <-- confidentiality control
public IActionResult GetSalary(int employeeId) { ... }
```

### ✅ Integrity — "Kya yeh data tampered toh nahi?"

Yeh wala log underestimate karte hain. Soch — agar attacker tera bank balance **padh** nahi sakta, but **change** kar sakta hai 5000 → 50,000 toh kaunsi cheez zyada khatarnak hai? Integrity attack zyada silent aur zyada damaging hota hai.

**Examples enterprise mein:**
- HMAC signatures on API payloads
- Database constraints + transaction logs
- Audit trails (immutable logs — append-only)
- Code signing (NuGet packages, Maven artifacts)
- Git commit signing (GPG / Sigstore)

**Backend analogy:** Jab tu apne microservice mein `ETag` aur `If-Match` headers use karta hai concurrent updates ke liye — woh integrity ki ek shakl hai (optimistic concurrency).

```http
PUT /orders/123 HTTP/1.1
If-Match: "v7"           # integrity check — only update if version matches
Content-Type: application/json
```

### ⚡ Availability — "Jab user ko chahiye, kya system mil raha hai?"

System ka up hona. **SLO 99.9%** wali baatein. DDoS attack? Availability attack hai. Ransomware? Availability + Integrity dono khatam.

**Examples enterprise mein:**
- Rate limiting (per-IP, per-token)
- Auto-scaling groups, multi-AZ deployments
- DDoS protection (Cloudflare, AWS Shield)
- Backups + tested restore procedures
- Circuit breakers (Polly in .NET, Resilience4j in Spring)

**Yaad rakh:** Availability sirf SRE ka problem nahi — yeh security ka problem hai. Backup encryption na ho toh confidentiality gayi. Backup test na ho toh availability gayi.

### 🧠 Mental Model #1 — "The Tripod"

CIA ek teen-tangaa stool hai. Ek bhi taang kamzor — sab gir gaya. Real-world mein har control trade-off banata hai:

| Tu Strengthen Karega | Kya Sacrifice Hoga |
|---|---|
| Confidentiality (heavy encryption) | Performance, sometimes availability |
| Integrity (HMAC + signatures) | Latency, complexity |
| Availability (open replicas) | Confidentiality risk badh sakta hai |

**Architect ka kaam:** balance dhoondhna based on **threat model + business context**. Banking app aur public blog ke CIA priorities same nahi hote.

### 🧠 Mental Model #2 — "Tag Every Requirement"

Future mein jab bhi feature spec aaye, tu mentally tag karega:

- "User profile photo upload" → C (private?), I (no tampering), A (CDN uptime)
- "Audit log write" → I-CRITICAL, C-medium, A-medium

Yeh habit Day 30 tak automatic ho jayegi.

## 3. Real-World Incident — Equifax (2017)

**Numbers:** 147 million Americans (almost half the US population), 15 million UK citizens. SSNs, DOBs, addresses, driver's licenses leaked. Settlement: **$700M+**. CEO resigned.

**Technical root cause:** Apache Struts 2 vulnerability **CVE-2017-5638** — a remote code execution flaw in the Jakarta Multipart parser. Patch March 7, 2017 release hua. Equifax ne apply nahi kiya. Attackers May 13 ko enter hue. **76 days inside the network**, undetected, before discovery on July 29.

### CIA lens se breakdown

- **Confidentiality FAILED** — 147M records exfiltrated
- **Integrity FAILED** — attackers could move laterally, modify data
- **Availability** — system up tha (ironic, the attack didn't crash anything — bilkul stealth)

### Architectural decisions jo prevent kar sakti thi

1. **Patch management process** — automated CVE scanning + SLA (critical = 72 hours)
2. **Network segmentation** — Struts-facing app ke peeche directly database access kyun thi? Trust boundaries missing.
3. **Egress monitoring** — 76 din mein TBs of data bahar gaya, koi alert nahi. Detective controls absent.
4. **Certificate expiry** — IDS device ka cert expire ho gaya tha 19 months pehle, isi liye encrypted traffic decrypt nahi ho raha tha. Pure operational hygiene failure.

### Lesson for the future architect

> *Security ek state nahi, ek process hai.* Patch nahi laga ke ya cert expire ho ke breach hote hain — fancy zero-days nahi. **Discipline beats brilliance.**
